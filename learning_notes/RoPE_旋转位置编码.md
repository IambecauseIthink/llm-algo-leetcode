# RoPE 旋转位置编码 学习笔记

> 围绕我的提问整理：频率从哪来、为什么要多个角速度、`xq_ * freqs_cis` 怎么严格对上、到底怎么衡量两个向量的距离。

---

## 0. 我最初的几个疑问

- `torch.outer(t, freqs)` 把位置和角速度配起来，矩阵会不会很大？每个位置对应多少个角速度？
- 为什么要有**不同**的角速度？
- 不同角速度配起来，最后还是可能转了多圈，那怎么衡量两个向量之间的距离？

下面从“RoPE 要解决什么”开始，逐个回答。

---

## 1. RoPE 要解决什么问题

注意力机制本身**不知道词的先后顺序**——把 “猫追狗” 和 “狗追猫” 的词打乱，attention 算出来一样。所以必须额外注入位置信息。

RoPE 的核心想法：**不要给向量“加”位置编码，而是把向量按位置“旋转”一个角度。**

- 第 0 个 token：转 0 度
- 第 p 个 token：转 p·θ 度

为什么旋转好？两个向量做点积（attention 算分数就是点积）时，**旋转后的点积只取决于角度差**，也就是**相对位置 `(p − k)`**。模型关心的是“两个词隔多远”，而不是各自的绝对位置。

$$\langle R_p\, q,\; R_k\, k \rangle = f(q, k,\; p-k)$$

---

## 2. 频率（角速度）从哪来

如果所有维度都用同一个角速度，不同位置很快转到重复角度（转一圈就回来），无法区分远距离。所以用**一组从快到慢的频率**：

```python
freqs = 1.0 / (theta ** (torch.arange(0, dim, 2).float() / dim))   # [dim//2]
# theta=10000, dim=64 → [1.0, ..., 0.0001 量级] 共 32 个角速度
```

$$\theta_i = \frac{1}{10000^{\,2i/d}}, \quad i = 0,1,\dots,\tfrac{d}{2}-1$$

- 第 0 对维度：角速度 = 1.0（转得最快，管局部/相邻关系）
- 最后一对维度：角速度 ≈ 0.0001（转得极慢，管长距离关系）

**关键：`dim` 个实数维度被两两配对，所以是 `dim//2` 个频率。** 每一对维度 `(x_{2i}, x_{2i+1})` 看成复平面上的一个点，共享一个角速度 $\theta_i$。

---

## 3. 位置 × 角速度 = 旋转角度（回答“矩阵会不会很大”）

```python
t = torch.arange(end)                  # 位置 [0,1,...,S-1]   形状 [S]
freqs = torch.outer(t, freqs)          # [S, dim//2]  位置 × 角速度 = 角度
freqs_cis = torch.polar(torch.ones_like(freqs), freqs)   # e^{i·角度}  [S, dim//2]
```

**纠正我最初的误解：**

- 角速度只有 `dim//2` 个（由**向量维度**决定），**只有一份，所有位置共享**。
- 不是“每个位置对应 end 个角速度”，而是“每个位置配 `dim//2` 个角速度”。
- 矩阵大小是 `S × (dim//2)`，比如 `2048 × 64`，不大，且**只算一次，全程复用**。

`outer` 的含义：

```
            θ_0     θ_1    ...   θ_31
位置 0      0·θ_0   0·θ_1        0·θ_31    → 都是0度，不转
位置 1      1·θ_0   1·θ_1        1·θ_31
位置 2      2·θ_0   2·θ_1        2·θ_31
```

`freqs_cis[p, i] = e^{i·(p·θ_i)}` = 第 p 个位置、第 i 对维度应旋转的复数因子。模长为 1（纯旋转），角度为 $p\theta_i$。

---

## 4. 为什么必须要“多个不同”的角速度（回答“转多圈”困惑）

假设只用**一个**角速度 θ：位置 0 → 0°，位置 p → p·θ°。

角度是**周期**的，转 360° 回到原点。若 θ=1°，位置 0 和位置 360 转到**完全相同**的角度——模型分不清。**单一频率必然周期性撞车，这是死结。**

解法像**时钟**：

```
时针(慢)：转一圈 12 小时   → 低频，管“大尺度”
分针(中)：转一圈 60 分钟   → 中频
秒针(快)：转一圈 60 秒     → 高频，管“小尺度”
```

光看秒针分不清 1 分 30 秒和 5 分 30 秒，但**三根针组合**的整体姿态在 12 小时内**永不重复**。

RoPE 一样：`freqs = [1.0, 0.1, 0.01, 0.001, ...]`

- 高频维度：转得快，相邻位置区分度高，但远距离会绕圈（像秒针）
- 低频维度：转得极慢（转一圈要上万个位置），整个序列内几乎不绕圈，负责“大概在序列前段还是后段”（像时针）

**任何单个频率都会撞车，但 `dim//2` 个频率组合的“整体旋转姿态”，在需要的序列长度内是唯一的。** 这和傅里叶用不同频率正弦波叠加表示任意函数是同一个数学思想。

---

## 5. 到底怎么衡量两个向量的距离（最关键）

**RoPE 衡量距离不是靠“看转了几圈”，而是靠 attention 里的点积。**

把一对维度看成 2D 平面向量。位置 $m$ 的 query 转了 $m\theta$，位置 $n$ 的 key 转了 $n\theta$，做点积：

$$\langle R(m\theta)\,q,\; R(n\theta)\,k \rangle$$

线性代数事实：$R(\alpha)^T R(\beta) = R(\beta-\alpha)$。所以：

$$\langle R(m\theta)q,\; R(n\theta)k\rangle = q^T R(m\theta)^T R(n\theta)\, k = q^T R\big((n-m)\theta\big)\, k$$

**右边：绝对位置 $m$、$n$ 消失了，只剩 $(n-m)$ —— 相对距离！**

含义：

- **不需要**去衡量“转了几圈”。
- 距离信息在 **query 和 key 点积的那一刻自动浮现**，体现为只依赖 $(n-m)$ 的旋转角。
- 单个频率会绕圈，但因为有多个频率、且 attention 只在乎点积结果，模型学到的是“每个频率上，相对距离 $(n-m)$ 对应一个固定的点积调制模式”，多个频率合起来把 $(n-m)$ 唯一编码。

一句话：**RoPE 不“测量”距离，它把距离编码进旋转；当 query 和 key 点积时，相对距离作为唯一的幸存者自动跳出来。**

## 6. 代码实现：把向量转成复数再旋转

### 6.1 query 转复数

```python
xq_ = torch.view_as_complex(xq.float().reshape(*xq.shape[:-1], -1, 2))
# [B,S,H,dim] → [B,S,H,dim//2]  复数
# 实数 [x0,x1,x2,x3,...] → 复数 [x0+x1·i, x2+x3·i, ...]
```

- `.float()`：`view_as_complex` 要求 fp32/fp64（bf16/fp16 会报错）。
- `.reshape(*xq.shape[:-1], -1, 2)`：把最后一维拆成“相邻两两一组”，`*xq.shape[:-1]` 是 Python 解包前面的维度。
- `view_as_complex`：要求最后一维为 2，把 `[实部, 虚部]` 合成复数，该维消失。

### 6.2 复数乘法 = 旋转，再转回实数

```python
freqs_cis = reshape_for_broadcast(freqs_cis, xq_)            # 补 1 维便于广播
xq_out = torch.view_as_real(xq_ * freqs_cis).flatten(3)
xk_out = torch.view_as_real(xk_ * freqs_cis).flatten(3)
```

- `xq_ * freqs_cis`：复数乘法，几何上 $z \cdot e^{i\theta}$ = 把 $z$ 旋转 $\theta$ 度。**这一个乘号是整个 RoPE 的心脏。**
- `view_as_real`：复数拆回 `[实部,虚部]`，多出长度为 2 的维度 `[B,S,H,32,2]`。
- `.flatten(3)`：从第 3 维起拍平，`32×2=64`，还原成输入形状 `[B,S,H,64]`。参数 3 是因为前面 B/S/H 三维要保留。
- query 和 key 用**同一个** `freqs_cis`，相对位置关系才能在点积里正确体现。

---

## 7. `xq_ * freqs_cis` 怎么“严格对上”（广播规则）

逐元素相乘时，PyTorch 从**最后一维往前**对齐：维度相等、或其一为 1（为 1 的被复制拉伸）。

旋转前形状对不上：

```
xq_:        [B, S, H, dim//2]   例 [2, 10, 8, 32]
freqs_cis:  [S, dim//2]         例    [10,    32]   ← 缺 B 和 H 维
```

所以先 `reshape_for_broadcast` 补 1：

```
freqs_cis:  [1, S, 1, dim//2]   例 [1, 10, 1, 32]
```

从右往左检查：

```
xq_:        [2, 10, 8, 32]
freqs_cis:  [1, 10, 1, 32]
最后一维：  32 == 32   ✓  每对维度配自己的旋转因子（一一对应不错位）
倒二维：     8 vs 1    ✓  1→复制8次：所有 head 用同一套位置编码
倒三维：    10 == 10   ✓  位置 S 严格对齐：第 p 个 token 配第 p 行旋转因子
最前维：     2 vs 1    ✓  1→复制2次：所有 batch 共用
```

**“严格对上”靠三点：**

1. **最后一维 `dim//2` 完全相等** → query 第 i 对维度乘的正是频率表第 i 个角速度对应的因子。
2. **S 维（位置维）相等且对齐** → 第 p 个 token 乘的正是 `freqs_cis[p]`，即它该转的角度。
3. B、H 维补成 1，靠广播复制 → 位置编码与 batch、head 无关。

如果 `reshape_for_broadcast` 把 1 补错位置（如 `[1,1,10,32]`），S 维会错位到 H 上、旋转因子全乱——这就是这个函数存在的意义。

---

## 8. 完整流水线（一张表串起来）

```
① freqs = 1/θ^(2i/d)        每个维度对“转多快”      [dim//2]
② t = arange(S)             有哪些位置             [S]
③ outer(t, freqs)           位置×角速度=该转的角度   [S, dim//2]
④ polar(1, ③) → freqs_cis   角度变成旋转复数 e^{iθ}  [S, dim//2]
⑤ view_as_complex(xq)       query 两维打包成复数     [B,S,H,dim//2]
⑥ reshape_for_broadcast(④)  给旋转因子补1好广播      [1,S,1,dim//2]
⑦ xq_ * freqs_cis           复数乘法 = 旋转!         [B,S,H,dim//2]
⑧ view_as_real + flatten    转回实数               [B,S,H,dim]
```

---

## 9. 三句话总结

- RoPE 用“旋转角度”编码位置，点积后自然只剩相对距离。
- 不同维度用不同角速度（高频管近、低频管远），角速度表就是 `freqs`；单一频率必然撞车，多频率组合保证唯一。
- 位置 × 角速度 = 旋转角度，变成复数后一个乘号完成旋转，广播保证位置和维度严格对齐。

---

## 10. 易错点备忘

- **配对方式有两种**：本 notebook 用 `reshape(...,-1,2)` 是“**相邻配对**” `(a0,a1),(a2,a3)`；HuggingFace LLaMA 用“**前后两半配对**” `(a0,a32),(a1,a33)`。两者 `freqs_cis` 排列不同、**不能混用**。
- `view_as_complex` 前必须 `.float()`，否则 bf16/fp16 报错。
- `freqs_cis` 预计算一次即可全程复用，不必每步重算。
- `flatten` 的参数跟着维度结构走：有 head 维是 `flatten(3)`，没有则 `flatten(2)`。
